Experience With Development Tools

Android SDK

I have gained an extensive understanding of the Android SDK throughout developing Android applications for personal projects. I have one major Android application release on the Google Play appstore (see Experience With Teams for link and teamwork details), which is a finance manager application. Specifically, the application involved the following:

  • Extending base Java view functionality, such as implementing a custom spinner with icons to adapt to the needs of the application
  • Implementing a custom side navigation menu, which has options that perform the required functionality
  • Extending the functionality of the standard action bar by adding additional buttons and dynamically changing the title
  • Making use of a 3rd party Calendar library (Caldroid) to build the main interface of the calendar to the project's needs. This also involved writing additional code to extend the Calendar to the personal needs of the project. The extra functionality, for instace, is displaying the amount of money spent/earned in one day on the Calendar.
  • Implementing charts to display statistics about the user's expenses and savings. This has been done using the MPAndroidChart library.
  • Implementing other 3rd party libraries to achieve various functionality, such as advertisements and an image gallery.
  • Creation and utilizing of a SQLite database to store the user's data and being able to access it efficiently
  • Writing the logic for the application itself (events, goals, tasks, recurring events, total expense/income calculation, etc.)
  • Input validation of the text input fields (involved regexp, such as disallowing to input "." as a numeric value, or allowing to input at most 2 numbers after the decimal point.

Java

JavaScript

Learned JavaScript in univesity as part of the Web & Database Technology course. I have built a Chess web application powered by Node.js (on the server) and JavaScript (on the client) in collaboration with another student. The project itself involved writing the entire Chess logic from scratch both on the client and the server. We have fully implemented the majority of chess rules, including movement, invalid move checking, checkmating, converting Pawn after it reaches the other side of the board and movement of the board. Visually, we have written the entire html from scratch and styled it with CSS, while also providing various functionality with clientside JavaScript (e.g. blinking tile in red when invalid move is made). We kept some logic on the serverside, such as pairing players for matches, returning responses for certain actions (such as end of match, move validity, etc.)
